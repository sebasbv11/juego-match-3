# Architectural Decision Records

Los ADRs registran decisiones técnicas relevantes, su contexto y sus consecuencias.

| ADR | Decisión | Estado |
| --- | --- | --- |
| [ADR-001](0001-arquitectura-modular.md) | Arquitectura modular por capas | Aceptada |
| [ADR-002](0002-autenticacion-clerk.md) | Autenticación administrada con Clerk | Aceptada |
| [ADR-003](0003-ranking-supabase.md) | Ranking diario en Supabase mediante API Node.js | Aceptada |
| [ADR-004](0004-despliegue-render-docker.md) | CI con GitHub Actions y CD en Render usando Docker | Aceptada |

Los nuevos ADRs deben usar numeración consecutiva y no deben reescribir una decisión histórica. Cuando una decisión cambie, se crea un ADR que sustituya al anterior.
